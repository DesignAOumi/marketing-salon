# プライバシー方針・データの取り扱い / Privacy & Data Handling

本ツールは **セルフホスト・持ち込みデータ方式** のOSSです。開発元は利用者の顧客データに一切アクセスしません。本書は設計上のデータ取り扱いと、運用者（利用者）の責任範囲を示します。

## データの所在

- 顧客データ・施術履歴・売上・予約などの全データは、利用者が運用するサーバー/PC 内（SQLite ファイル + アプリコンテナ）にのみ保存されます。
- 開発元・第三者のサーバーへは一切送信・保管されません。
- リポジトリには顧客データ・DBファイル・`.env`・鍵を含めません（`.gitignore` + CI 検査）。

## テレメトリ

- 利用状況の収集・送信（テレメトリ、アナリティクス、クラッシュレポート、自動アップデート確認等）は **一切行いません**。
- ネットワーク通信は、利用者が明示的に有効化した外部連携（Anthropic API / Google Calendar）のみに限定されます。

## AIアドバイスと外部送信

- **既定は「連携なし（オフライン）」**: 110件の定型アドバイスを顧客データにローカル照合します。外部送信ゼロ。
- **連携あり（Claude BYO・任意）**: 利用者自身の Claude APIキー使用時のみ作動。設定の **送信フィールド・ホワイトリスト** に含めた項目だけを送信し、氏名・連絡先等の PII は **既定で送信しません**。氏名が必要な文面でも、既定で **匿名化**（`{{NAME}}` 置換 → ローカルで実名に再差込）します。
- 送信内容は監査ログに記録できます（本文 PII は保存しない設定）。

## 暗号化

- Claude APIキー等の機微フィールドは `ENCRYPTION_KEY` を用いて AES-256-GCM で **保存時暗号化** します。
- ログインパスワードは bcrypt でハッシュ化（復号不可）。

## 同意・保持・削除（個人情報保護法 / GDPR 配慮）

- **連絡同意 `consentToContact`**: 既定 false。再来店提案・連絡文生成・一括連絡は同意=true の顧客のみが対象。撤回は即時反映。
- **保持期間**: 既定「最終来店から3年」でアーカイブ候補として通知（設定で変更可）。自動物理削除は既定OFF。
- **削除権**: 顧客単位の論理削除 → 物理削除に対応（復元不可の確認を要する）。
- **開示・ポータビリティ**: 顧客データを CSV / JSON でエクスポート可能。

## 運用者の責任範囲

セルフホストの性質上、データの保護・バックアップ・アクセス管理・法令遵守の責任は運用者にあります。公開ネットワーク設置時は HTTPS 化と強固な秘密鍵管理を必ず行ってください（詳細は [SECURITY.md](../SECURITY.md)）。
